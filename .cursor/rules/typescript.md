# TypeScript Rules (ATE9)

## 型安全ポリシー
- `any` は原則禁止。  
  どうしても必要な場合は理由コメントを残し、`unknown` やジェネリクスで代替できないか検討する。
- `strict` オプションを前提とした実装を行う。
- 可能な限り **型推論** に任せつつ、公開 API やコンポーネントの props には明示的な型を付ける。

## 型の管理
- ドメインに関わる型は `@/domain/*` などの専用ディレクトリにまとめる。
- Supabase 由来の型は生成ツールまたは `Database` 型などから参照し、  
  「手書きの型」と二重管理しない。
- API レスポンス型や DTO は `@/types/api` 等にまとめ、  
  コンポーネント内で ad-hoc に定義しない。

## コーディングスタイル
- `Promise<void>` ではなく可能な限り意味のある戻り値を返す。
- `null` と `undefined` の扱いは意識して区別する。
- 再利用可能なユーティリティは `@/lib/*` に切り出す。
