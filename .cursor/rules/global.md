# ATE9 Global Rules

## プロジェクト概要
- プロジェクト名: **ATE9**
- LP と Admin からなる Web プロジェクト。
- デザインのソース・オブ・トゥルースは  
  `.cursor/ATE9_designdesign/design.html` と `design.png`。
- 実装時はまずこれらを確認し、**レイアウト・余白・タイポ・カラー**を極力合わせること。

## デザインの原則
- ベースは **Mercari 公式サイト** 風の「白ベース / 余白多め / ミニマル」。
- アクセントカラーは **Electric Blue `#497CFF`**。  
  赤系アクセントは基本使わない。
- セクションごとに背景トーンを変える（白 / ごく薄いグレー / ごく薄いブルー）。
- 見出しは太字サンセリフ、本文は読みやすさ優先。
- 角丸はカード・ボタンともに **16〜20px** を基本とする。
- UI コンポーネントは **shadcn/ui** をベースに構築（詳細は `ui.md`）。

## 実装スタイル
- 言語は **TypeScript**。`any` は禁止（詳細は `typescript.md`）。
- コンポーネント・ロジックは **再利用性と責務の分離**を意識する（詳細は `architecture.md`）。
- Supabase との通信は専用レイヤー経由で行う（詳細は `supabase.md`）。
- アニメーションは必要最小限・上品に。Mercari の動きに寄せる（詳細は `animation.md`）。

## 開発フローでの AI の使い方
- 新規実装・リファクタ時は、まず `design.html` を見ながら「どのセクションに対応するコードか」を明示する。
- 既存コード修正時は「どのルールファイルに関わる変更か」を明記し、それを満たす形に修正する。
- ルールに迷ったら **保守性・可読性・一貫性** を優先する。
