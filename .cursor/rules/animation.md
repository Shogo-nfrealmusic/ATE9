# Animation Rules (ATE9)

## 基本方針
- ベースは **静的に綺麗** であること。アニメーションはあくまで補助。
- Mercari 公式サイト同様、
  - フェードイン
  - 軽いスライド
  - 透明度とスケールの小さな変化
  が中心。激しい動きは避ける。

## 使用ライブラリ
- 適宜、以下を使って良い：
  - [Aceternity UI](https://ui.aceternity.com/)
  - Framer Motion
  - GSAP
- セクション単位の入場アニメーションには Framer Motion を優先的に使用する。
- 複雑なタイムラインやスクロール連動が必要な場合のみ GSAP を検討。

## 実装ルール
- `motion.div` 等を乱立させず、セクションのラッパーにまとめる。
- 1つのセクションにつき、主要なアニメーションは 1〜2 種類に絞る。
- パフォーマンスを意識し、`layout` ではなく `transform` を使ったアニメを優先。

## Mercari のアニメーションへの寄せ方
- Hero やカードの切り替えは、
  - フェード
  - スライド（X 方向 20〜40px 程度）
  - イージングは `easeOut` / `easeInOut` 系を使用
- ホバー時：カードやボタンは
  - ほんの少しのスケールアップ（1.02〜1.05）
  - シャドウの追加
  程度に抑える。
